# 系统架构设计

## 核心原则

1. **引擎层与产品层分离** - 确保 70%+ 代码可复用
2. **API 优先** - 所有模块定义清晰接口
3. **可插拔设计** - 新产品可快速组装

---

## 引擎层模块详解

### 1. 用户中心 (User Center)

**职责**: 所有产品的统一身份认证

```
┌─────────────────────────────────────┐
│            用户中心 API              │
├─────────────────────────────────────┤
│ POST /auth/register   # 注册        │
│ POST /auth/login      # 登录        │
│ POST /auth/logout     # 登出        │
│ GET  /user/profile    # 获取资料    │
│ PUT  /user/profile    # 更新资料    │
│ GET  /user/membership # 会员状态    │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  存储：PostgreSQL (users 表)        │
│  缓存：Redis (session/token)        │
└─────────────────────────────────────┘
```

**可复用性**: ⭐⭐⭐ 所有产品直接调用

---

### 2. 支付中心 (Payment Center)

**职责**: 统一处理订阅、订单、支付回调

```
┌─────────────────────────────────────┐
│            支付中心 API              │
├─────────────────────────────────────┤
│ POST /payment/create-order          │
│ POST /payment/callback/wechat       │
│ POST /payment/callback/alipay       │
│ GET  /payment/order/:id             │
│ GET  /payment/subscription/status   │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  集成：微信支付、支付宝、Stripe     │
│  存储：PostgreSQL (orders 表)        │
└─────────────────────────────────────┘
```

**可复用性**: ⭐⭐⭐ 所有付费产品都需要

---

### 3. 内容 CMS (Content Engine)

**职责**: 通用的内容管理和发布系统

```
┌─────────────────────────────────────┐
│           内容 CMS API               │
├─────────────────────────────────────┤
│ GET  /content/courses               │
│ GET  /content/courses/:id           │
│ POST /content/lessons               │
│ PUT  /content/lessons/:id           │
│ GET  /content/media/:id             │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  存储：PostgreSQL + MinIO/S3        │
│  搜索：Elasticsearch (可选)         │
└─────────────────────────────────────┘
```

**可复用性**: ⭐⭐ 教育类产品直接复用，其他产品可适配

---

### 4. 数据分析平台 (Analytics)

**职责**: 统一埋点、事件追踪、报表生成

```
┌─────────────────────────────────────┐
│          数据分析 API                │
├─────────────────────────────────────┤
│ POST /analytics/track               │
│ GET  /analytics/dashboard           │
│ GET  /analytics/user/:id/events     │
│ GET  /analytics/conversion          │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  存储：ClickHouse / TimescaleDB     │
│  可视化：Grafana / 自研看板          │
└─────────────────────────────────────┘
```

**可复用性**: ⭐⭐⭐ 所有产品都需要数据驱动

## 技术栈建议

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | React + TypeScript | 组件化、类型安全 |
| 后端 | Node.js (NestJS) 或 Python (FastAPI) | 快速开发、生态丰富 |
| 数据库 | PostgreSQL | 关系型数据首选 |
| 缓存 | Redis | 高性能缓存 |
| 消息队列 | RabbitMQ | 异步任务处理 |
| 容器 | Docker + K8s | 标准化部署 |
| 监控 | Prometheus + Grafana | 可观测性 |
