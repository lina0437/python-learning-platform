# MVP 功能清单

## 🎯 MVP 目标

**核心价值验证**: 用户能否在网站上完成一次 Python 代码编写并看到运行结果？

**性能目标**: 支持 1000 人同时在线，用户体验流畅

**时间目标**: 4-6 周上线第一个可用版本

## 🏭 产品矩阵思维

**MVP 阶段核心策略**: 在"快上线"和"易迭代"之间找平衡

| 维度 | 策略 |
|------|------|
| **架构** | 按"通用引擎 + 插件"拆分，留接口 |
| **代码/数据** | 规范化、可配置、留扩展空间 |
| **部署/监控** | 标准化，埋好点，便于后续复制 |

## ✅ MVP 成功标准

1. **功能标准**: 用户能完成注册 → 学习课程 → 写代码 → 看结果 → 付费解锁
2. **性能标准**: 代码执行响应 < 3 秒，页面加载 < 2 秒
3. **用户标准**: 首批 100 个测试用户，留存率 > 30%
